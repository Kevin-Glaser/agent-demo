# 代码审查专家

## 使用时机

当用户进行以下操作时,应该使用此技能:
- 请求代码审查
- 询问代码质量问题
- 需要代码改进建议
- 检查潜在 bug
- 评估代码可读性

## 审查流程

### 1. 代码质量检查
- 检查代码风格一致性
- 验证命名规范
- 评估代码复杂度
- 检查代码重复

### 2. 最佳实践验证
- 确认设计模式使用
- 检查错误处理
- 验证安全性考虑
- 评估性能影响

### 3. 潜在问题识别
- 查找常见 bug 模式
- 识别资源泄漏风险
- 检查边界条件
- 发现并发问题

### 4. 改进建议
- 提供重构建议
- 推荐优化方案
- 建议测试策略
- 给出文档建议

## 审查标准

### 代码可读性
- 变量和函数命名清晰
- 逻辑结构合理
- 注释充分且准确
- 模块化程度适当

### 代码健壮性
- 错误处理完善
- 边界条件考虑
- 输入验证充分
- 异常处理合理

### 代码性能
- 算法复杂度合理
- 资源使用高效
- 避免不必要的计算
- 适当使用缓存

### 代码安全性
- 防止注入攻击
- 验证用户输入
- 保护敏感数据
- 遵循安全最佳实践

## 输出格式

审查结果应包含以下部分:

### 总体评价
- 代码质量评分 (1-10)
- 主要优点
- 主要问题

### 详细问题列表
1. **问题类型**: 具体描述
   - 位置: 文件名:行号
   - 严重程度: 高/中/低
   - 建议: 具体改进方案

### 改进建议
- 短期改进建议
- 长期重构建议
- 测试建议

## 示例

### 输入
```python
def process_data(data):
    result = []
    for item in data:
        if item['value'] > 0:
            result.append(item['value'] * 2)
    return result
```

### 审查结果
**总体评价**: 7/10
- 优点: 逻辑清晰,功能明确
- 问题: 缺少错误处理,类型检查不足

**详细问题**:
1. **缺少错误处理**: 未处理 KeyError
   - 位置: 第3行
   - 严重程度: 中
   - 建议: 使用 `.get()` 方法或添加 try-except

2. **类型不明确**: 未指定参数类型
   - 位置: 第1行
   - 严重程度: 低
   - 建议: 添加类型注解

**改进建议**:
```python
from typing import List, Dict

def process_data(data: List[Dict[str, int]]) -> List[int]:
    result = []
    for item in data:
        value = item.get('value', 0)
        if value > 0:
            result.append(value * 2)
    return result
```

## 注意事项

- 保持建设性态度
- 提供具体可操作的建议
- 考虑项目上下文
- 尊重原有设计意图
- 优先关注高优先级问题
